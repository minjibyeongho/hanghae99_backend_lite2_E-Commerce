# 캐시

## 1. 개요
- 구성된 프로젝트에서 캐시를 적용이 필요한 부분에 대해서 Redis 캐싱 전략 활용 구현

## 2. 이슈사항 도출
### 2-1. 인기 상품 TOP5
- 강의에서 예시로 말씀해주신 것 처럼 집계에 대해서 DB 자원을 계속해서 사용하기 보다는 캐시로 돌리는 것이 좋을 것으로 판단됨
- 집계 쿼리로 부하가 높음
- ?? 현재 진행하는 프로젝트(1인 진행)에서는 사용자가 많이 않아서 얼마나 차이가 나는지 모르겠음..

### 2-2. 상품 상세
- 상품 상세 정보는 조회 빈도수가 높을 것으로 예상됨

## 3. 적용 방안
### 3-1. key 설계
- 2-1. 인기 상품 TOP5 -> sales:top-products:last3days
- 2-2. 상품 상세 -> product:{productId}:detail
- ?? TTL은 얼마가 적정할지 잘 모르겠어서 AI로 조사한 내용을 토대로 각각 10분, 1시간 적용
